Multimodal LLM

以下介绍一些典型的模型和架构：

Vision Language Action

RT-1

指令和视觉输入，输出动作。

由于末端使用 Transformer，将 action离散化为 256 种，也就是 Transformer 输出的 Token

FiLM：将 language Condition 到 images 上，其中β \betaβ﻿ 和 γ\gammaγ﻿ 是 encode 后的 language。

EfficientNet：更具一定的规则、利用 Scalling law 扩大视觉的模型

Token learner：输入为 feature map，提取出重要的 Token，学习出重要性权重

通过在 RT-1 的训练数据中混入仿真数据，模型在仿真中效果提升，但是在 Real 中基本没有下降。Data diversity 是比绝对的数据量更重要的。

RT-2

基于预训练的 VLM，针对 Vision-language 与 Robot-Action Data 共同微调。

效果比 RT-1 更好，通过众多的互联网 Vision ，language data 实现泛化能力。引入 Chain of Thought，对于复杂任务先做 planning。

RT-X

Vision 和 language 预训练是必要的；模型的参数量更大更容易实现 Cross-Embody。

RT-H：Action hierarchies

引入 language motion，预测原子动作，将语义无关的任务关联起来，实现 data sharing。不同形态硬件动作可能不同，但是 language motion 可能是相同的，提高了可解释性。

人类可以在训练过程中不断纠正 language motion。

RT-2 将动作编码为一种 token，通过De-tokenize 将这几个 token 转化为 ΔT\Delta TΔT﻿, ΔR\Delta RΔR﻿，通过 ik 解出 Δθ\Delta \thetaΔθ﻿

实现跨体的泛化只需要训练一个 tokenizer（因为不同的设备执行相同的任务，语义层面是相似的）。RT-2 的预训练模型为 VLM，能很好地理解语义。由于 Action 模态的数据不够，动作做的不够精确。由于未采集放置数据，所以放置时可能会推动物体。

采数据的效率：1000/16h

遥操作：面对抖动，可能需要滤波。

100万条数据，各种机器人，任务。但是数据的质量较低，因为相机的位置、数目不规范。

Transfussion，使用独立的 Action expert

Flow matching：针对每个点做条件 flow matching

在特定的任务和机器人上 fine-tune，通过过拟合这个任务，提高在这个任务上的表现。

但是微调后出现了灾难性遗忘。同时泛化能力比较差，如果 Test time OOD，微调效果很差。

对比 flow match 和 diffusion：

核心目标：两者都旨在学习一个从简单先验分布（通常是标准高斯噪声）到复杂数据分布的转换过程；它们都通过定义一个“前向过程”逐步破坏数据（加噪），然后学习一个“反向过程”从噪声中重建数据（去噪）。

核心差异：关键区别在于前向过程的性质以及如何参数化和学习反向过程。

GraspVLA

使用大量合成数据，加上少量真机数据微调。如果训练很充分，通过少量数据微调就可以在一个小类别上实现泛化。

OpenVLA

OpenVLA 目标实现 Cross-Embodiment，但是具体实现较难。

使用 SigLIP 代替 CLIP：把 Softmax 替换为 Sigmoid（SigLIP（Sigmoid Loss for Language-Image Pre-training）采用二分类sigmoid损失函数替代CLIP的多分类softmax损失函数，从而解决了CLIP在训练效率、显存占用和数值稳定性等方面的局限性。）

DINO & DINOv2：通过自监督学习计算机视觉的特征

以上工作的特点

通过Auto-regressive 只预测下一个 Token：BC 中误差可能会累积；human demonstration 并不稳定

改进：使用 action chunk；使用生成式模型建模多峰分布

Action chunk：预测多步

ACT：Action Chunking Transformer

CVAE：Conditional Variational Auto-encoders

Variational Autoencoder, VAE 的一种扩展形式。cVAE 在 VAE 的基础上引入了条件变量，使得生成的样本能够受到外部条件的控制。

Diffusion Policy：应用 DDPM

Navigation

目前实现效果不错，因为仿真和真实世界的差异不是很大，多数情况下没有碰撞，所以没有 physics Gap。由于训练好的 VLM 已经见过了大量的仿真和真实的视频，所以容易克服这种 GAP。

Uni-Navid

有多种任务：到某个房间；找到某个物体；探索房间后回答问题。

indoor navigation 实际上是一种特殊的自动驾驶。

Future work

  1. 研究如何通过新的数据采集范式（如人类演示视频）来获取新技能。
  1. 理解预训练数据集应该如何构建。与 NLP 类似，预训练阶段用什么数据决定模型泛化能力。要研究：数据多样性、任务分布、模态比例等对性能的影响。
  1. 了解达到近乎完美性能需要多少数据，以及需要什么类型的数据。
  1. 多源数据会不会互相干扰？还是能促进泛化？这是 foundation model 的核心问题之一。
  1. 构建更大、更高质量的数据集；探索更高效的训练方法或更高效的数据来源。
  1. 持续构建用于机器人操控的 VLA 基础模型。
  1. 利用仿真数据
  1. 在基础模型上使用 RL 做 fine-tuning、习得更复杂或更长期的策略，提升在现实任务中的实际表现。
