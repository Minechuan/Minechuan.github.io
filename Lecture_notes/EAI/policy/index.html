开环的规划的动态性不足，为了实现闭环的、动态调整的方式，我们希望学习一个策略。

Imitation Learning

基础知识

状态迭代执行：在MDP 中当前的决策与之前的状态无关，但是实际过程中可能要考虑历史的信息。

但是这个思路不一定能 work，因为存在分布的偏移，即专家的轨迹与模仿的轨迹存在偏差。即现有的状态与模仿学习中的状态有较大差异，不知道该如何做。事实上：step 越长偏差越大。

例子：在无人驾驶汽车中需要不断纠正错误，即快要碰撞或驶出道路时，让专家（人类驾驶员）进行纠正，这些新的数据可以帮助我们学到的策略更好地回到专家数据的分布。另外：使用额外的两个相机进行数据增强，如果正面的相机看到了左侧的相机的图像，需要向右微调。

经典的 BC：遥操作。基于动作捕捉，基于机械臂的主从臂来实现操控。

相比于合成数据：不能保证视觉或者物理层面的GAP。相反，进行遥操作和真机数据就可以不必考虑这些 GAP。

在虚拟环境下遥操作

Human-in-the-Loop Imitation Learning using Remote Teleoperation. 在虚拟环境中发生一些assets和位置的置换，得到更多数据。好处是更容易获取更多的数据。

Why might we fail to fit the expert?

A.Non-markovian behavior

一些情况下data 并不是马尔科夫的。比如“老司机”的预判，并不容易学习。解决：使用历史信息，比如利用 RNN；Casual confusion：在观测的信息高维时，模型可能搞错了因果关系。即自动驾驶误以为是在减速的观测下 action 为踩刹车。

B.action 的多峰分布

需要多峰的表征来刻画，不预测 action，而是去预测 action 的分布。这里可能需要使用更复杂的 backbone 网络。

解决方式

1️⃣ Output mixture of Gaussians

使用多个高斯分布来拟合动作空间的分布。

2️⃣ Latent variable models

3️⃣ Diffusion Models

4️⃣ Autoregressive discretization

对于 action 的多峰分布不易使用 regressive直接拟合，这里对高维动作空间中动作的每一个维度进行回归。

Multi-task learning

在多任务情况下我们的策略见过了更多的场景，可能能够更好地应对 Policy 运行过程中出现的偏差。但是 multi-task 也需要考虑 goal 的多峰分布。

Imitation Learning 总结

Behavior cloning 的数据来自第三方，并没用环境之间交互的过程，训练的效果很大程度上取决于训练数据的分布。所以又称作 Offline learning(离线学习)。

而 reinforcement Learning是 Online Learning.

Normalization / Scaling

优势函数归一化（Advantage Normalization）：公式：Atnorm=At−μAσA2+ϵA_t^{norm}= \frac{A_t - \mu_A}{\sqrt{\sigma_A^2 + \epsilon}}Atnorm​=σA2​+ϵ​At​−μA​​﻿

  • 仅在训练时使用
  • 在小批量（minibatch）中进行归一化
  • 当价值函数尚未训练充分时，归一化可以帮助 AtA_tAt​﻿ 以 0 为中心

状态归一化（State Normalization）：公式：stnorm=st−μrunσrun2+ϵs_t^{norm}= \frac{s_t - \mu_{run}}{\sqrt{\sigma_{run}^2 + \epsilon}}stnorm​=σrun2​+ϵ​st​−μrun​​﻿

  • 需要保存状态的运行均值，并在测试时使用
  • 类似于 Batch Normalization，有助于加速训练

奖励缩放（Reward Scaling）：公式：rtscale=rtσrun2+ϵr_t^{scale}= \frac{r_t}{\sqrt{\sigma_{run}^2 + \epsilon}}rtscale​=σrun2​+ϵ​rt​​﻿

  • 奖励缩放使得价值函数 Vφ(st)V_\varphi(s_t)Vφ​(st​)﻿ 的训练更加容易

Initialization & Activation

正交初始化 (Orthogonal Initialization)：

  1. 从标准高斯分布初始化权重矩阵
  1. 对初始化的 W 使用 SVD，W=UTDVW=U^TDVW=UTDV﻿，然后令 W=V。正交初始化效果更好。

使用 Tanh activation：

Tanh 作为激活函数比 ReLU 小狗更好。

适用于浅层 MLP（如 2-3 个隐藏层），梯度消失问题不显著。

PPO 总结

  • 优势：
    • 通过Importance Sampling 和 minibatch sampling，采样效率高
    • 通过 Clips，KL约束，训练稳定，表现好。
    • 比 TRPO 数学简单。
  • 劣势：
    • 采样效率不如 off-policy RL
    • 有许多针对特定问题的 tricks，需要精调超参数。
